# Decision Log

## D-001 — Worldtree is a separate first-party ecology client

- Date: 2026-07-14
- Decision: Use a separate top-level folder and repository for Worldtree while
  retaining Intersectionalities of Becoming as the constitutional kernel.
- Reason: game experience and ontology governance have different rhythms and
  failure modes.
- Rejected: monorepo coupling, copied ontology, and replacement-project framing.

## D-002 — The new Worldtree repository is canonical

- Date: 2026-07-14
- Decision: `Uploaded-Intelligence/Worldtree-Civilisation-of-Becoming` and the
  matching local folder are the sole active game home.
- Consequence: the uncommitted Diegetic scaffold remains untouched as a
  superseded archive.

## D-003 — Solo value precedes federation

- Date: 2026-07-14
- Decision: the game must be useful to one Player before network effects.
- Consequence: the Playable Seed tests a complete solo artifact-to-growth loop.

## D-004 — Occurrence grows a trace; ratification grows meaning

- Date: 2026-07-14
- Decision: artifact capture may immediately grow a trace bud, while interpreted
  branches remain spectral until Player ratification.
- Consequence: satisfying game feel does not collapse evidence into truth.

## D-005 — Worldtree is the primary space; other maps are lenses

- Date: 2026-07-14
- Decision: Constellation and ecological-dynamics views later project the same
  lineage rather than creating separate models.

## D-006 — Model use is routed and subordinate

- Date: 2026-07-14
- Decision: deterministic code owns policy and state. Luna handles bounded
  volume after evaluation, Terra handles complex synthesis, and Sol high/xhigh
  is reserved for exceptional design and review.
- Consequence: no blanket strongest-model default and no model self-escalation.

## D-007 — Separation stops before product implementation

- Date: 2026-07-14
- Decision: this campaign creates the durable repository handoff only.
- Consequence: no application scaffold, runtime dependency, API integration, or
  playable feature is introduced from the Intersectionalities task.

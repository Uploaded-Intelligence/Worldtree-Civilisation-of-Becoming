# Architecture and Ownership Boundary

Status: separated foundation; planned runtime components do not exist.

## Ecology topology

```text
Intersectionalities of Becoming
constitutional ontology · consent · standing · authority · provenance
                         │
            released versioned contracts
                         │
                         ▼
Worldtree Civilisation of Becoming
solo play · Campaign Trees · artifacts · possibility · game feel
                         │
                 future bounded acts
                         │
                         ▼
consensual resonance and federation (later gate)
```

## Repository ownership

Worldtree owns game-specific domain language, projections, interaction design,
local artifact experience, rendering, accessibility, and Player-facing
explanations. It may express upstream concepts through metaphor, but every such
projection must disclose what it reveals and suppresses.

Intersectionalities owns the semantic meaning of events, claims, consent,
standing, mandate, authority, provenance, contestation, revocation, and
federation. Worldtree must consume released contracts through an immutable
manifest containing the ontology version, source tag, source commit, and file
digests.

## Planned playable data flow

```text
Player horizon
      │
lived artifact → trace bud (occurrence only)
      │
deterministic or bounded possibility hypothesis
      │
spectral branch → inspect evidence and uncertainty
      │
Player ratifies or refuses
      │
durable branch projection or archived refusal
```

Only the separation/control-plane layer exists today.

## Planned technical direction

- Node 24, Next.js 16, React, TypeScript, and accessible SVG.
- Local-first append-only event lineage and derived projections.
- IndexedDB artifact vault; no backend during the Playable Seed.
- Deterministic possibility grammar first; model adapter later.
- Constellation and ecological-dynamics maps as later lenses over the same
  lineage, not competing data models.

These are ratified constraints for the next campaign, not claims of implemented
software.

## Trust boundaries

- lived occurrence versus interpretation;
- Player articulation versus model hypothesis;
- spectral possibility versus durable ratified growth;
- local artifact bytes versus explicitly authorized disclosure;
- game projection versus upstream semantic contract;
- model capability versus authority.

## Rollback

Separation does not modify the legacy Diegetic folder or the upstream kernel.
The Worldtree foundation can be reverted through Git. Future local data formats
must be versioned and exportable before incompatible migrations.
